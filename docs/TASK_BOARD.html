📜 Правила управления задачами

**Создание задач**: Только STRATEGY может добавлять новые задачи

**Изменение статусов**: Roo обновляет статус после подтверждения выполнения

**Переход к IN_PROGRESS**: Roo запрашивает подтверждение перед стартом

**Готовность к TESTING**: После завершения работы Roo помечает READY

**Merge в DONE**: После review и merge STRATEGY помечает DONE

🔄 Workflow

STRATEGY → [TASK_PACK] → ROO → [выполнение] → STRATEGY → [review] → DONE
