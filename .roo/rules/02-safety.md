## Safety Rules

### Требуют явного подтверждения "подтверждаю":
- docker compose down (остановка всего стека)
- docker compose restart bot (рестарт бота)
- изменение .env файла
- DROP TABLE, DELETE FROM без WHERE
- rm -rf любой директории
- git reset --hard

### Запрещено без отдельного обсуждения:
- TRADING_MODE=live
- изменение RISK_MAX_DRAWDOWN
- удаление данных из таблиц trades

### Workflow (обязательно для каждой задачи):
1. Прочитать все затрагиваемые файлы полностью
2. Изложить план (не более 5 шагов)
3. Ждать подтверждения плана от пользователя
4. Выполнять пошагово — отчитываться после каждого шага
5. НЕ переходить к следующему шагу без подтверждения
6. Summary: что сделано, риски, что дальше
