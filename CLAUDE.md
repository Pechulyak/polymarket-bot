# Протокол оркестрации (polymarket-bot, S1)
- Primary source rule: любой вердикт исполнителя — гипотеза до проверки по живому
  коду/БД с file:line. Схему таблиц не предполагать — проверять запросом.
- Money-adjacent (push, restart, DDL, ордера): СТОП, запросить "confirmed"
  у оператора. Deny-правила это дублируют — не обходить.
- Изменение crontab — только по паттерну: crontab -l > backups/crontab_bak_<YYYYMMDD_HHMM>.txt →
  новый crontab собрать файлом в repo → показать оператору diff старый/новый →
  после "confirmed" установить crontab <файл>. Прямое crontab -e и установка
  без бэкапа и diff запрещены.
- Делегирование: стандартный dev-код → scripts/mm.sh "<ТЗ>" 30. Тривиальное —
  сам. Архитектура, деньги, секреты — сам, не делегировать.
- ТЗ исполнителю самодостаточно: файлы, критерии приёмки, команда проверки.
  Исполнитель не видит нашу переписку.
- Цикл отказов: M3 попытка 1 → fail → M3 попытка 2 (уточнённое ТЗ) → fail →
  субагент debugger → fail → СТОП, отчёт оператору. Без исключений.
- После возврата: тесты → субагент reviewer → только потом приёмка.
- Секреты в вывод не включать. Публичные адреса и api_key UUID — можно.
- git add только явными именами файлов. push — только оператор.
- S2 (62.60.233.100) не трогать: SSH S1→S2 заблокирован, фарминг-деплой вне сессии.
- Финальный отчёт по задаче: что сделано, кем (M3/Opus/сам), попыток, diff-статистика.
- Завершение любой backlog-задачи (TASK_ID) обязательно включает документацию:
  статус в docs/TASK_BOARD.md, строка в changelogs/CHANGELOG.md (формат: дата |
  TASK_ID | суть, без логов и кода), при изменении архитектуры/файлов системы —
  PROJECT_STATE.md. Объёмные doc-блоки (PROJECT_CHANGELOG) — делегировать
  исполнителю через mm.sh, короткие записи (1-5 строк) — напрямую. Задача не
  считается DONE без документации. Финальный отчёт включает список обновлённых
  docs.
- Задачи оператор ставит краткой формулировкой. Развёртывание в план, ТЗ
  исполнителю, контроль качества и итерации — моя зона без промежуточных
  согласований. К оператору выхожу только: confirmed-гейты, финальный отчёт,
  неразрешимый блокер после полного цикла отказов.
